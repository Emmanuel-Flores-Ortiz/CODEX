# 🎮 Nombre del Proyecto 🎮

Guildex.

---

## ✨ Descripción

Este proyecto fue desarrollado como parte de una práctica enfocada en los temas vistos en Repositorios.
Es un sistema de administración de un Gremio para videojuego RPG dentro de C++, con una infraestructura de repositorio.

---

## ⬇️ Instalación

El proyecto funciona mediante CMake; se puede compilar en entornos como Visual Studio o CLion.

---

## 🛠️ Tecnologías Utilizadas


* Control de versiones: `Git & GitHub`
* Creacion de Códigos: `C++, CMake`
* Programación: `CLion, Notepad++`

---

## 📂 Estructura del Proyecto

```
Módulo 1 — `members.cpp`
Gestión del roster de aventureros. Registro, baja, búsqueda y listado de miembros.

Módulo 2 — `inventory.cpp`
Gestión del inventario del gremio. Alta de objetos, búsqueda, listado y valor total.

Módulo 3 — `quests.cpp`
Gestión de misiones. Registro de quests, asignación a miembros y marcado como
completadas.

Módulo 4 — `interface.cpp` y `main.cpp`
Menú principal que integra los tres módulos anteriores. Navegación y reportes
generales.

- Funciones con parámetros y valor de retorno
- Estructuras (struct) para Adventurer, Item y Quest definidas en guildex.h
- Ciclos (for, while o do-while)
- Condicionales (if-else)
- Selección múltiple (switch-case)
- Asignación dinámica de memoria con malloc, calloc y realloc
- Liberación de memoria con free al terminar el programa

Estructura de branches obligatoria:

  main                  rama principal, siempre debe compilar
  feature/members       rama del módulo 1
  feature/inventory     rama del módulo 2
  feature/quests        rama del módulo 3
  feature/interface     rama del módulo 4

```
